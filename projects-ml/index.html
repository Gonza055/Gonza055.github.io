Selected Projects

Predictive Maintenance – Sensor Data Conditioning & Feature Engineering (2025)

Context. Industrial maintenance setting with 50k+ high-noise sensor time-series from crushing and grinding equipment.

What I built

  • Preprocessing pipeline for noisy sensor data (smoothing, outlier capping, signal reconstruction).
  • Reliability-focused feature sets (load ratios, temperature deltas, transient-spike metrics).
  • Structured datasets ready for downstream failure-prediction models.

Tech stack

  • Python, Pandas, NumPy, Jupyter.
  • Time-series preprocessing, feature engineering, EDA.

Impact

Enabled future ML experimentation for early-warning indicators and reliability analysis, and designed reliability features for downstream ML failure prediction.

TrainOps Simulation Data Optimization (2024)

Context. Data-engineering project using ~200k rail-simulation records for transit-planning scenarios.

What I built

  • Python & C++ parsers to convert raw simulation logs into structured, analysis-ready datasets.
  • Automation workflow that reduced output-processing time by ~70%.
  • Pipelines integrating rider-survey data with onboard sensor logs to validate scenarios.

Tech stack

  • C++, Python, Pandas, basic SQL, Git.

Impact

Improved scenario comparison and supported data-driven decisions for service configuration and capacity planning.

ML Failure-Mode Prototyping (2024–25)

Context. Academic project inspired by industrial predictive-maintenance work.

What I built

  • ML pipeline with cleaning, denoising, feature engineering, and exploratory labeling of failure modes.
  • Baseline models (Random Forest, SVM, Gradient Boosting).
  • Evaluation using standard metrics plus visualization of feature effects and error patterns.

Tech stack

  • Python, scikit-learn, Pandas, NumPy, Matplotlib.

Outcome

Provided a prototype workflow for predicting equipment behavior and a sandbox for experimenting with reliability-oriented ML models.
